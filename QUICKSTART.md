# T.C.H. 聊天室快速开始指南

## 🚀 立即体验

直接访问：[T.C.H. 聊天室](https://bjx-pc-z.github.io/tch-chat/)

## 📱 基本使用

### 1. 设置仓库
- 点击右上角设置按钮
- 输入你的 GitHub 仓库：`用户名/仓库名`
- 保存设置

### 2. 发送消息
- 在左侧消息输入框中填写标题和内容
- 可选择添加标签
- 点击发送消息

### 3. 浏览消息
- 查看右侧消息列表
- 使用搜索框快速查找
- 点击消息卡片跳转到 GitHub

## ⚙️ 配置说明

### API 密钥（可选）
为了获得更好的性能和更高的请求限制：

1. 访问 [GitHub Personal Access Tokens](https://github.com/settings/tokens)
2. 创建新的 token，勾选 `repo` 权限
3. 在设置页面中输入 token

**无密钥限制**：
- 每小时 60 次请求
- 仅可访问公开仓库

**有密钥优势**：
- 每小时 5000 次请求
- 可访问私有仓库
- 更快响应速度

## 🏗️ 部署到你的仓库

### 方法一：直接 Fork（推荐）

1. 点击页面右上角的 "Fork"
2. 修改 `src/App.tsx` 中的默认仓库配置
3. 推送到你的仓库
4. 启用 GitHub Pages

### 方法二：下载源码

1. 下载整个项目源码
2. 修改配置中的仓库信息
3. 使用 GitHub Actions 部署

## 🛠️ 自定义配置

### 修改默认仓库
编辑 `src/App.tsx` 文件：

```typescript
const config = {
  owner: '你的GitHub用户名',
  repo: '你的仓库名',
  // ... 其他配置
};
```

### 调整刷新频率
在设置中可以调整自动同步间隔：
- 10秒 - 实时更新
- 30秒 - 推荐设置
- 60秒 - 节能模式

## 🔧 故障排除

### 常见问题

**Q: 页面显示"暂无消息"**
A: 检查仓库配置，确保仓库存在且有写入权限

**Q: 发送消息失败**
A: 确认仓库允许创建 Issues，或添加 API 密钥

**Q: 页面加载缓慢**
A: 添加 GitHub API 密钥可显著提升性能

**Q: 搜索不到消息**
A: 尝试使用简单的关键词搜索
